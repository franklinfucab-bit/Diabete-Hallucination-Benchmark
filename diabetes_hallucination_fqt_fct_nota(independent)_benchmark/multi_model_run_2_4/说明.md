# 糖尿病幻觉基准测试

用于评估大语言模型（LLM）在糖尿病医学知识上的表现，以及其抵抗幻觉、虚假自信和误导性前提的能力。

## 概述

本基准包含三种互补的测试类型，分别针对医疗 AI 的不同失效模式：

| 测试类型 | 缩写 | 题目数 | 说明 |
|----------|------|--------|------|
| **虚假问题测试** | FQT | 1,000 | 在 plausible 临床情境中嵌入虚构概念。正确答案需质疑虚假前提。 |
| **虚假自信测试** | FCT | 962 | 标准多选题，仅有一个正确答案。测试事实准确性与过度自信。 |
| **以上皆非** | NOTA | 1,000 | 所有选项均错误；正确答案为「以上皆非」。测试对看似合理但错误选项的抵抗力。 |

## 文件格式

### 基准文件

- **FQT、FCT 与 NOTA**：JSONL（每行一个 JSON 对象）

每题对象通常包含：
- `id`：唯一标识符
- `question`：题目文本
- `options`：选项数组，含 `option_id`、`text`、`is_correct`
- `correct_answer`：正确选项 ID
- `ground_truth`：正确答案的解释
- `metadata`：生成信息（模型、时间戳、主题等）

### 评估报告

JSONL 格式：
- 首行：汇总（总题数、得分、成功/失败数）
- 后续行：每题评估，含质量分数与反馈

## 使用方法

1. 加载对应测试类型的基准文件。
2. 将题目与选项输入模型。
3. 将模型所选答案与 `correct_answer` 比对。
4. 可选：使用评估报告获取质量指标与逐题分析。

## 评估指标

- **FQT**：concept_fidelity、sycophancy_trap、authority_bias、precision_of_refusal、difficulty_discrimination
- **FCT**：technical_accuracy、cognitive_trap_design、difficulty_discrimination、domain_relevance、test_effectiveness
- **NOTA**：technical_accuracy、cognitive_trap_design、difficulty_discrimination、domain_relevance、test_effectiveness
